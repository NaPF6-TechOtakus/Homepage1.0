### 步骤三：核心交互功能开发

- **操作内容**：为页面添加时间相关的动态功能和用户交互组件。
- **使用的工具**：Deepseek V4 PRO + Trae 内置 Agent
- **具体做法**：
  1. **动态问候语**：根据 `new Date().getHours()` 判断时间段，在 5-12/12-17/17-22/22-5 区间分别显示 Good morning/afternoon/evening/night

- **结果**：页面具备了时间感知能力，导航交互流畅自然，高光块位置精确不偏移。

## 遇到的问题与解决方案

### 问题一：页面整体偏左，未能居中显示

- **问题现象**：页面初始版本使用 `margin: 0 auto` 居中 `.board-inner`，但在某些屏幕宽度下整体偏左。
- **解决过程**：使用 AI 分析问题，发现绝对定位的容器内 `margin: auto` 不可靠。解决方案是将 `.main-board` 改为 `display: flex; justify-content: center`，利用 Flexbox 的弹性居中能力。
- **最终结果**：页面在所有屏幕宽度下完美居中。

### 问题二：导航菜单高光块上下错位

- **问题现象**：鼠标移到菜单项上时，白色高光胶囊块与菜单文字错位，没有准确覆盖目标行。
- **解决过程**：通过 AI 调试分析，发现初始代码使用了 `element.offsetTop`，该属性返回的是相对于 `offsetParent` 的距离，而不同元素的 `offsetParent` 可能不同（取决于定位方式），导致坐标体系不一致。解决方案是改用 `getBoundingClientRect()` 获取相对于视口的绝对坐标，用 `itemRect.top - menuRect.top` 计算相对偏移。
- **最终结果**：高光块精确对齐每个菜单项，滑动平滑不抖动。

### 问题三：紧凑导航栏高光写死在"关于我"上

- **问题现象**："学习分享"和"我的作品"点开后，左上角紧凑导航栏的高光仍然停留在"关于我"按钮上。
- **解决过程**：分析发现 `active-item` class 写死在 HTML 的"关于我"按钮上（`class="compact-nav-item active-item"`），没有动态切换机制。解决方案是给每个按钮添加 `data-nav` 属性，创建 `setCompactActive(name)` 函数，在 `openAbout()` / `openStudy()` / `openWorks()` 中调用，关闭时切回 `setCompactActive('home')`。
- **最终结果**：紧凑导航栏高光跟随当前页面状态正确切换。

### 问题四：视频卡片从长条列表改为网格布局

- **问题现象**：初始版本"我的作品"页面中，6 个视频以纵向长条排列，视觉效果单调且浪费空间。
- **解决过程**：用户要求改为横排网格 + 4:3 预览缩略图。使用 `flex-wrap: wrap` + `width: calc(33.333% - 11px)` 实现每行 3 个的响应式网格。为每张卡片添加 `aspect-ratio: 4 / 3` 的预览区，内含渐变背景 + 居中播放按钮 + 左上角分类标签。重新设计卡片结构为"上部预览区 + 下部信息区"。
- **最终结果**：视觉效果类似视频网站的作品集展示，信息密度合理。

### 问题五：照片墙拖动与单击放大的冲突处理

- **问题现象**：需要同时支持鼠标拖动照片和单击放大照片，两种操作容易冲突。
- **解决过程**：在 `mousedown` 事件中同时监听 `mousemove` 和 `mouseup`。如果在 `mouseup` 之前鼠标移动距离超过 3px（`Math.abs(dx) > 3 || Math.abs(dy) > 3`），标记 `dragged = true` 并更新位置；否则在 `mouseup` 时判定为单击，触发 `zoomPhoto(src)` 放大。
- **最终结果**：拖动和单击两种交互互不干扰，体验流畅。

### 问题六：时钟数字因比例字体导致左右晃动

- **问题现象**：时钟时间每秒/每分钟刷新时，数字位置轻微左右晃动，因为 Georgia 是比例字体，"1"的宽度比"0"窄。
- **解决过程**：将 `font-family` 改为等宽字体栈：`"SF Mono", "Cascadia Code", "Consolas", "Courier New", monospace`，确保所有数字字符宽度一致。
- **最终结果**：时钟显示稳定，无晃动。

### 问题七：多个弹窗状态同时存在导致页面重叠

- **问题现象**："关于我"、"学习分享"、"我的作品"三个详情页可能同时显示，导致页面混乱。
- **解决过程**：在 `openAbout()`、`openStudy()`、`openWorks()` 三个打开函数中，先互相清除其他状态（`classList.remove('study-active', 'works-active')` 等），确保同时只有一个 `body` 状态类。CSS 层面也添加互斥规则（如 `body.about-active .study-detail { opacity: 0; pointer-events: none }`）。
- **最终结果**：三个详情页状态互斥，不会重叠。

### 问题八：照片墙逐张渲染时图片加载时序不可控

- **问题现象**：照片在 `img.onload` 回调中定位，但不同图片大小差异导致加载完成时间不一，需要确保前一张渲染完成后再显示下一张。
- **解决过程**：在 `onload` 回调内使用 `setTimeout(..., idx * 350)` 固定延迟序列，配合 CSS `transition-delay` 实现视觉上的逐张出现效果。每张照片入场时从 `scale(0.3)` 弹到正常大小，间隔 350ms，营造"一张一张飞入"的节奏感。
- **最终结果**：照片依次弹入，动画节奏统一，不受网络加载速度影响。

## 总结与反思

- **本次作业中，我学到了**：
  - HTML/CSS/JavaScript 三层分离的网页开发模式：HTML 负责结构，CSS 负责样式，JavaScript 负责行为
  - CSS Flexbox 弹性布局的居中、换行、间距控制
  - CSS `transition` 过渡动画的核心原理：属性变化时自动补间，配合 `cubic-bezier` 自定义缓动曲线
  - `position: absolute` + `left`/`top` 精确控制桌面端卡片位置
  - `body` class 状态驱动的页面切换模式：通过添加/移除 class 触发不同的 CSS 规则集
  - 等宽字体 vs 比例字体的区别及其在数字显示场景中的选择
  - `navigator.clipboard.writeText()` 浏览器剪贴板 API
  - `getBoundingClientRect()` 获取元素屏幕坐标，解决不同层级间的相对定位计算
  - Canvas 2D 绘图基础：渐变、模糊滤镜、动画循环
  - `aspect-ratio` CSS 属性实现固定宽高比的响应式预览区

- **AI 工具对我的帮助主要体现在**：
  - 快速生成复杂的 HTML/CSS/JS 代码：从 Canvas 背景到照片墙拖拽系统，AI 能直接产出可运行的代码
  - 精准调试定位问题：当遇到"高光错位"、"数字晃动"等具体 bug 时，AI 能快速分析根因并提供精确修复
  - 渐进式功能迭代：每次只需描述"要什么效果"，AI 自动处理 CSS 动画、JS 交互、状态管理的全部细节
  - 代码审查与错误排查：能一次性检查整个文件的引用一致性，发现孤立的 CSS 规则和冲突的 class 引用
  - 技术支持讲解：能将每一步修改涉及的技术知识点系统化讲解，帮助理解底层原理

- **我认为 AI 工具的局限性在于**：
  - AI 的"记忆"有限，长对话后可能遗忘早期修改，需要反复确认上下文
  - 视觉效果的微调（如颜色、间距、动画速度）需要多次"感觉不对→再调"的迭代，AI 只能执行指令无法预判审美偏好
  - 对于较复杂的布局（如绝对定位的卡片系统），AI 的初始代码可能不完美，需要人工提出具体的改进方向
  - 复杂的交互逻辑（如拖动+单击区分）需要明确的规则定义，否则 AI 倾向于简化处理

- **如果重新做一次，我会**：
  - 更早确定所有卡片的功能和位置，减少后期频繁调整布局（如从"推荐分享"到"学习分享"到"我的作品"入口的变更）
  - 先规划好状态管理模式（body class 驱动），再逐个开发详情页，避免后期补状态互斥逻辑
  - 尝试引入 CSS Grid 替代部分绝对定位，使响应式适配更优雅
  - 为视频文件生成封面缩略图而非纯色渐变背景，提升作品卡片视觉吸引力
  - 在照片墙中加入更多交互：双指缩放、右键菜单、图片懒加载等
